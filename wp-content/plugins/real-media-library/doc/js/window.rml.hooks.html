Methods

(static) call(name, args, contextopt)

Execute functions hooked on a specific action hook.
Parameters:
Name Type Attributes Description
name string The name of the event. Just have a look at the doc events or create your own.
args Array.<mixed> The arguments passed to the callback
context object <optional>
 The context passed to the callback

(static) exists(name)

Checks if a specific action hook exists.
Parameters:
Name Type Description
name string The name of the event. Just have a look at the doc events or create your own.

(static) register(name, callback)

Hooks a function on to a specific action.
Parameters:
Name Type Description
name string The name of the event. Just have a look at the doc events.
callback function The callback with the arguments passed by the .call function. The last argument is jQuery ($)

Events

action/$action

Action buttons are buttons in the folder details or in the RML option page. With this hook you can modify the post data for a specific action. $action can be replaced with the data-action attribute of the button.
This:
  • jQuery
Properties:
Name Type Description
post object The post data

action/done/$action

Action button is pressed and the response of server is available. $action can be replaced with the data-action attribute of the button.
This:
  • jQuery
Properties:
Name Type Description
post object The post data
response object The response of the server

attachments/move

Move or copy attachments to a folder. This hook is called before server request.
This:
  • allInOneTree
Properties:
Name Type Description
ids Array.<int> The attachment ids to move
folderId string The destination folder id
isShortcut boolean True if the attachments are copied to the destination folder

attachments/moved

Attachments are moved or copied to a folder.
This:
  • allInOneTree
Properties:
Name Type Description
ids Array.<int> The attachment ids to move
folderId string The destination folder id
isShortcut boolean True if the attachments are copied to the destination folder

attachments/relocate

The user wants to relocate an attachment within a folder. This hook is called before the request to the server.
Properties:
Name Type Description
folderObj jQuery The a-tag folder object
attachmentId int The attachment id which is relocated
nextId int The new attachment id next to attachmentId
lastId int The last id of the visibile attachments (if pagination has no reached all pages yet)
event event The drag event
ui object The drag event ui

attachments/relocated

The user has relocated an attachment within a folder.
Properties:
Name Type Description
folderObj jQuery The a-tag folder object
data object The data sent to the server
response object The response from the server

folder/created

A folder is created and available in the tree view.
This:
  • allInOneTree

folder/relocated

The folder hierarchy is changed and a folder is relocated. The arguments passed to this event are the same as the callback function of allInOneTree "onRearrangeRelocate".
This:
  • allInOneTree

folder/removed

A folder is removed.
This:
  • allInOneTree
Properties:
Name Type Description
id string The folder id

folder/renamed

A folder is renamed.
This:
  • allInOneTree
Properties:
Name Type Description
folderId string The folder id
newName string The new name for the folder
slug string The new slug for the folder

general

This first fired event. It is not similar to jQuery(document).ready().

grid/request

Modify the request for a folder. Arguments passed to the hook are same as jQuery.ajaxPreFilter callback retrieves.

grid/request/$type

Modify the request for a folder type. Arguments passed to the hook are same as jQuery.ajaxPreFilter callback retrieves.

grid/toolbar

Before the grid toolbar gets initialized.

grid/toolbar/default

Modify the default start up folder.
This:
  • wp.media.view.AttachmentFilters.RML
Parameters:
Name Type Description
filters object The filters for the attachment toolbar
Example

Initially load another folder for startup

window.rml.hooks.register("grid/toolbar/default", function(filters) {
	this.model.set(filters["my-slug"].props, { silent: false } );
});

tree/afterInit

This hooks is called after a tree is initialized.
This:
  • allInOneTree

tree/afterInit/$listMode

This hook is called after a tree is initialized in a given view. $listMode can be replaced with "grid" or "list".
This:
  • allInOneTree

tree/afterInit/ML

This hook is called after a tree is initialized in the default media library page of wordpress backend (upload.php).
This:
  • allInOneTree

tree/afterInit/ML/$listMode

This hook is called after a tree is initialized in the default media library page of wordpress backend (upload.php) for a given view. $listMode can be replaced with "grid" or "list".
This:
  • allInOneTree

tree/afterInit/modal

This hook is called after a tree is initialized in the "Insert media"-dialog.
This:
  • allInOneTree

tree/custom

This hook is fired for every new custom list created with PHP function wp_rml_select_tree().
Parameters:
Name Type Description
object jQuery The tree

tree/custom/rml-list-$inputName

This hook is fired for a specific new custom list created with PHP function wp_rml_select_tree(). The $inputName can be replaced with the first parameter $inputName of wp_rml_select_tree() function.
Parameters:
Name Type Description
object jQuery The tree

  • jQuery
Properties:
Name Type Description

tree/prepare/aioSettings/$listMode

The tree is not initialized yet through the allInOneTree plugin. Here you can modify the options for the jQuery plugin. The $listMode can be replaced by 'grid' or 'list'.
This:
  • jQuery
Properties:
Name Type Description
aioSettings object The settings for the jQuery plugin

tree/prepare/aioSettings/modal

The tree is not initialized yet through the allInOneTree plugin. Here you can modify the options for the jQuery plugin. This hook is explicit called in a modal window ("Insert media"-dialog).
This:
  • jQuery
Properties:
Name Type Description
aioSettings object The settings for the jQuery plugin

tree/prepare/ML

A tree container is placed to the DOM in the Media Library page of wordpress backend (upload.php). But the tree is not initialized yet through the allInOneTree plugin.
This:
  • jQuery

tree/prepare/modal

A tree container is placed to the DOM in a modal view ("Insert media"-dialog). But the tree is not initialized yet through the allInOneTree plugin. The context passed to the hook functions is the media modal window object (.media-modal).
This:
  • jQuery
Properties:
Name Type Description
container jQuery The container
isAlreadyLoaded boolean The modal window gets re-opened
customSelectToChange string The query selector for the backbone select dropdown of the RML filter
